# DS studio

DS studio 是一個 Chrome 擴充功能，專為優化 `chat.deepseek.com` 的使用體驗而設計。它讓您可以建立**多組提示詞組**，在發送訊息前自動將所選的提示詞注入到輸入內容中，並針對不同場景快速切換。

## 主要功能

- **多組提示詞管理**：建立、編輯、重新命名、刪除多組提示詞組，針對不同場景（程式碼審查、翻譯、寫作等）快速切換。自訂下拉選單支援搜尋篩選與拖曳排序。提示詞內容透過鉛筆按鈕在獨立的 1280×720 編輯視窗中編輯，享有寬敞的編輯空間並維持自動儲存。
- **自動提示詞注入**：在使用者按下 Enter 鍵或點擊發送按鈕時，自動將當前選取的提示詞附加到訊息最前方。
- **全域預設提示詞**：設定一段在所有對話中都會自動附加的提示文字，與各對話的提示詞組獨立運作。配有專屬開關可單獨停用全域提示詞注入（主開關優先權最高 — 主開關關閉時無論此開關狀態為何皆不注入），內容同樣透過鉛筆按鈕在獨立編輯視窗中編輯。
- **對話綁定提示詞組**：每個對話可以獨立綁定不同的提示詞組，切換對話時自動還原該對話使用的提示詞組。
- **頁面內快速切換 Overlay**：在 DeepSeek 對話頁面頂端標題列中央直接顯示提示詞組下拉選單，無需開啟彈出選單即可快速切換提示詞組，且與彈出選單雙向同步。行動版（< 768px）採用 gap 模式定位於按鈕之間，並內建自動穩定機制：頁面載入時持續校正位置直到版面完全安定，確保重新整理後不會卡在錯誤的早期量測值。
- **空選項模式**：下拉選單永遠提供空白選項，選取後不注入任何提示詞組，適合不需要提示詞的對話。
- **回到頂部按鈕**：在對話頁面右下角自動提供與原生「回到底部」按鈕外觀一致的「回到頂部」浮動按鈕；適用於長對話快速返回起始處，且自動整合至 Markdown 匯出流程以確保完整擷取對話。
- **行動裝置側欄滑動手勢**：在行動裝置上，於畫面中央 80% 區域內向右滑動即可展開或收合側邊欄，解決行動版缺乏側邊欄快速切換的問題。手勢辨識包含滑動距離、方向主導性、持續時間三重防誤觸機制，並自動避開 Chrome Android 系統返回手勢區域。
- **雲端同步**：提示詞組、對話綁定與設定會自動透過 `chrome.storage.sync` 在登入相同 Chrome 帳號的所有裝置間同步。彈出選單標題列會即時顯示同步狀態（綠色「雲端同步」或紅色「未同步」），並提供「手動同步」按鈕可隨時將本機資料重新推送至雲端。
- **同步衝突處理**：當偵測到多台裝置間的雲端資料不一致時，系統會主動提示衝突，並提供「合併同步」功能，防止資料被靜默覆蓋。
- **智慧提示詞合併**：匯入 JSON 備份或處理同步衝突時，系統會基於提示詞 ID 進行去重合併，並自動保留最新修改（依據 `updatedAt` 時間戳記）的內容。
- **JSON 備份與還原**：可將所有提示詞組與介面設定匯出為 JSON 檔案，或從 JSON 備份還原設定（提示詞組合併、UI 設定覆寫）。另提供「匯出／匯入復原備份」與「清除所有已還原紀錄」按鈕，可獨立備份、還原與清除「被審查回覆」的還原記錄。
- **引用回覆**：在 AI 回覆區域中選取文字後，畫面上方自動出現「引用回覆」浮動按鈕；點擊後將選取內容以 Markdown blockquote 格式（`> 內容`）附加至輸入框，支援多行選取（每行各自加上 `> ` 前綴）。
- **恢復被審查的回覆**：當 DeepSeek 以「我暫時無法回答這個問題」等訊息取代原本的模型回覆（內容審查）時，擴充功能會自動從串流資料中還原原始回覆並顯示於畫面上，附「⚠ 已復原內容」徽章；若有思考過程也會一併重建為可折疊區塊。還原記錄自動保存（最多 200 筆），重新整理頁面或重新開啟對話後仍可自動恢復。
- **隱藏思考過程**：啟用後，DeepSeek 思考區塊（推理過程）在出現時自動收合，讓畫面更簡潔；已手動展開的區塊不受影響。
- **側邊欄自動隱藏**：滑鼠離開側邊欄時自動收合至 60px 寬度，滑鼠移入時展開，節省畫面空間。
- **對話區域寬度調整**：可自由調整對話訊息區域的寬度（30%–100% 視口寬度），適應不同閱讀偏好。
- **編輯輸入框寬度調整**：可獨立調整輸入框的寬度（30%–100% 視口寬度），且輸入框寬度會自動受對話區域寬度限制（不超過對話區域寬度）。
- **對話匯出**：提供一鍵匯出功能，可將目前 DeepSeek 聊天室中的對話紀錄**完整**匯出為 Markdown (`.md`) 格式檔案。即使是超長對話中尚未顯示在畫面上的訊息，也會透過自動滾動擷取完整收集；擷取期間您仍可繼續輸入與發送訊息，畫面會顯示進度提示。若擷取逾時或被手動捲動中斷，系統仍會匯出已收集的部分內容並在檔尾附上警告。
- **思考過程開關**：可選擇是否在匯出的 Markdown 中包含 AI 的思考過程（Thinking Process）。
- **參考連結開關**：可選擇是否在匯出的 Markdown 中包含搜尋參考連結（例如 `[link-1]`）。
- **系統時間注入**：啟用後，每則發送的訊息開頭會自動加上目前系統時間（`Current Time: yyyy/mm/dd hh:mm:ss`），讓 AI 知道當下的真實時間。
- **即時自動儲存與同步**：所有設定變更（切換提示詞組、編輯內容、開關狀態、UI 調整）都會即時儲存到瀏覽器本地與雲端同步空間（Sync）。
- **主開關連動**：右上角主開關關閉時，所有子功能（側邊欄自動隱藏、隱藏思考過程、系統時間注入、寬度調整、頁面內 Overlay 選單等）會一併停用，確保一鍵關閉所有擴充功能行為。
- **快速開關**：內建啟用／停用開關，可依需求隨時切換注入功能，而不影響原有設定。
- **舊版資料遷移**：從舊版升級時，原有的提示詞內容會自動遷移到「我的提示詞」組別。

## 安裝說明

### 透過 Chrome 擴充功能頁面載入（開發者模式）

1. 下載或複製本專案至本機。
2. 開啟 Chrome 瀏覽器，在網址列輸入 `chrome://extensions/` 並前往。
3. 開啟右上角的「**開發人員模式**」（Developer mode）開關。
4. 點擊左上角出現的「**載入未封裝項目**」（Load unpacked）按鈕。
5. 在彈出的檔案選擇視窗中，選取 `ds-studio` 資料夾（包含 `manifest.json` 的根目錄）。
6. 載入成功後，擴充功能會出現在列表中，並在瀏覽器工具列顯示圖示。

### 安裝後注意事項

- 安裝完成後，建議重新整理已開啟的 `chat.deepseek.com` 分頁，以確保 Content Script 正確載入。
- 如果擴充功能圖示未出現在工具列，可點擊右側的拼圖圖示，找到 DS studio 並釘選。

## 使用說明

### 管理提示詞組

1. 點擊瀏覽器右上角的擴充功能圖示開啟選單。
2. （可選）點擊「Global Prompt」區域中「全域預設提示詞」旁的**鉛筆按鈕**，在開啟的獨立編輯視窗（1280×720）中輸入您希望在所有對話中自動附加的文字，編輯內容會自動儲存。卡片右側的開關可單獨控制全域提示詞是否注入。
3. 在「Prompt Group」自訂下拉選單中選取要使用的提示詞組，或保持空白選項以不使用任何提示詞組。下拉選單支援：
   - **搜尋篩選**：輸入關鍵字即可即時過濾提示詞組名稱。
   - **拖曳排序**：拖曳提示詞組左側的拖曳把手即可調整順序。
4. 下拉選單右側有兩個按鈕，展開下拉選單後各提示詞組亦有懸停操作按鈕：
   - **+**（下拉選單右側）：新增提示詞組（會跳出命名對話框，名稱為必填）。
   - **鉛筆**（下拉選單最右側）：開啟獨立編輯視窗，編輯當前選中提示詞組的內容；未選中任何提示詞組時此按鈕停用。
   - **✎**（懸浮於提示詞組右側）：重新命名該提示詞組。
   - **✕**（懸浮於提示詞組右側）：刪除該提示詞組（需確認）。
   - 系統允許刪除所有自訂提示詞組，下拉選單會永遠保留一個空白選項作為預設。
5. 在獨立編輯視窗的文字區域編輯提示詞內容，編輯內容會即時自動儲存；重複點擊鉛筆按鈕會聚焦既有的編輯視窗而非開啟新視窗。
6. 開啟右上角的啟用開關，擴充功能即會生效。
7. 不同對話可各自綁定不同的提示詞組：切換對話時，下拉選單會自動切換到該對話綁定的提示詞組。
8. 不同分頁（Tab）各自獨立：每個瀏覽器分頁可獨立設定不同的提示詞組，互不影響。新分頁開啟新對話時不會繼承其他分頁的提示詞選取。
9. （可選）勾選或取消勾選「匯出包含思考過程」與「匯出包含參考連結」來控制 Markdown 匯出行為。
10. （可選）勾選「注入系統時間」，讓每則發送的訊息開頭自動附上目前系統時間。

### 使用 UI 調整功能

1. 在彈出選單的「UI Adjustments」區域，可分別調整以下設定：
2. **側邊欄自動隱藏**：勾選後，DeepSeek 左側邊欄會在滑鼠離開時自動收合至 60px 寬度，滑鼠移入時展開。
3. **隱藏思考過程**：勾選後，DeepSeek 的思考區塊（推理過程）在出現時會自動收合；使用者手動展開的區塊不受影響。
4. **對話區域寬度**：開啟切換開關後，透過滑桿調整對話訊息的顯示寬度（30%–100%）。
5. **編輯輸入框寬度**：開啟切換開關後，透過滑桿獨立調整輸入框的顯示寬度（30%–100%）。注意：輸入框寬度會自動受「對話區域寬度」限制，不會超出對話區域。
6. 當右上角主開關關閉時，以上 UI 調整功能會一併停用。

### 使用提示詞注入

1. 前往 `chat.deepseek.com`，正常輸入您的訊息並發送。
2. 擴充功能會在背景自動將當前選取的提示詞（以及全域預設提示詞）加入訊息前方。
3. 隨時可透過下拉選單切換提示詞組，無需重新整理頁面。
4. 不同對話可獨立綁定不同的提示詞組 — 系統會根據當前對話的 UUID 自動還原您上次為該對話設定的提示詞組。
5. 不同瀏覽器分頁（Tab）的提示詞組設定互不影響 — 每個分頁獨立記憶當前選取的提示詞組。
6. 新對話第一次發送訊息後，系統會自動將當前選取的提示詞綁定到新產生的對話 UUID。

### 不使用任何提示詞組

1. 在「提示詞組」下拉選單中選取空白選項。
2. 此時提示詞組的鉛筆編輯按鈕會呈現停用狀態，此為正常現象 — 系統不會注入任何提示詞組內容。
3. 若您有設定「全域預設提示詞」且其開關為開啟狀態，該內容仍會被注入。

### 匯出對話

1. 在 `chat.deepseek.com` 頁面中，點擊擴充功能圖示。
2. 按下「匯出當前頁面對話為 Markdown」按鈕。
3. 系統會先自動捲動至對話頂端，再由上而下逐步擷取完整對話內容（包含長對話中尚未顯示在畫面上的訊息），畫面上會顯示目前擷取進度。
4. 擷取期間您仍可正常輸入與發送訊息，但**請勿手動捲動對話記錄**，以免擷取中斷。
5. 擷取完成後，系統會自動下載一份 `.md` 檔案到您的電腦，並還原您原本的捲動位置。
6. 若擷取逾時或被手動捲動中斷，系統仍會匯出已收集的部分內容，並在檔案末尾附上「內容可能不完整」的警告，不會讓您空手而歸。

### 恢復被審查的回覆

此功能是完全自動的，無需任何設定：

- **運作方式**：當 DeepSeek 將原本的模型回覆替換為「我暫時無法回答這個問題」等審查訊息時，擴充功能會自動還原原始回覆並顯示在畫面上，附「⚠ 已復原內容」徽章。
- **思考過程**：若原始回覆包含思考過程，也會重建為可折疊的思考區塊並顯示思考時間。
- **跨重整保留**：還原記錄會自動保存在本機（最多 200 筆），重新整理頁面或之後再回到該對話時，被審查的訊息仍會自動恢復顯示。
- **備份管理**：可在彈出選單的「Backup & Restore」區域匯出／匯入還原記錄備份，或一鍵清除所有已還原紀錄。

### 行動裝置側欄滑動手勢

此功能僅在行動裝置上生效，無需任何設定：

- **觸發方式**：在 DeepSeek 對話頁面正中央 80% 區域（扣除上下左右各 10% 邊界）內，用手指向右滑動即可展開或收合側邊欄。
- **防誤觸**：系統會自動辨識滑動方向與距離——僅有明顯的向右滑動（≥ 50px、水平為主、< 500ms）才會觸發，垂直捲動或短暫觸摸不會誤觸。
- **相容性**：觸發區域刻意避開螢幕左右邊緣，避免與 Chrome Android 的系統返回手勢衝突。
- **無需設定**：此功能隨擴充功能主開關自動啟用/停用，無獨立開關。

### 回到頂部按鈕

「回到頂部」按鈕是完全自動的，無需任何設定：

- **外觀**：與 DeepSeek 原生的「回到底部」按鈕外觀完全一致（圓形、相同顏色與邊框），僅箭頭方向相反（向上）。
- **出現時機**：當您向下捲動對話，離開第一則訊息後，按鈕會在右下角原生按鈕的上方（或相同位置）自動出現。
- **使用方式**：點擊按鈕即可自動捲動至對話最頂端；也可使用鍵盤（Enter 或 Space）。捲動過程中按鈕維持可點，**再次點擊即可在當前位置停止**捲動（不會重新開始）。
- **隱藏時機**：當您已位於對話最頂端時，按鈕自動隱藏，不會佔用畫面空間。
- **無需設定**：此功能隨擴充功能主開關自動啟用/停用，無獨立開關。

### 備份與還原設定

1. 在彈出選單的「Backup & Restore」區域：
2. **備份**：點擊「備份設定（匯出 JSON）」按鈕，系統會下載包含所有提示詞組與設定的 JSON 檔案。
3. **還原**：點擊「還原設定（匯入 JSON）」按鈕，選擇先前備份的 JSON 檔案。
   - 還原時：UI 設定（全域提示詞、寬度、開關等）會被覆寫；提示詞組則會以 ID 為基礎進行合併，相同 ID 保留較新版本，新 ID 的提示詞組會新增。
   - 對話綁定（chatPresetMap）同樣會進行合併。
4. **復原備份**：「匯出復原備份」與「匯入復原備份」按鈕可單獨備份／還原「被審查回覆」的還原記錄（獨立於一般設定備份）；「清除所有已還原紀錄」則可一鍵刪除這些記錄。
5. **手動同步**：點擊「手動同步」按鈕可立即將本機資料重新推送至雲端同步空間，適合在標題列顯示「未同步」時使用。

### 雲端同步衝突處理

1. 若多台裝置同時編輯提示詞組，當擴充功能偵測到雲端與本地資料不一致時：
2. 下次開啟彈出選單時會顯示「雲端同步衝突」對話框。
3. 點擊「合併同步」後，系統會以 ID 為基礎合併兩邊的提示詞組，保留最新修改。
4. 介面設定則以雲端版本覆寫本機版本。

## 開發與安裝

此專案採用 Manifest V3 開發，並符合 Chrome 擴充功能的官方安全規範。所有 DOM 攔截邏輯皆運作於 Content Script 中，確保注入的內容能正確觸發目標網站的狀態更新。支援 `chrome.storage.sync` 跨裝置同步，並內建衝突偵測與智慧合併機制。
