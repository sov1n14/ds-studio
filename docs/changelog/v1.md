## [1.10.0] - 2026-05-22

### Added
- **Quote Reply** (`content/quote-reply.js`): New floating button feature that appears when the user selects text inside `div.ds-virtual-list-visible-items` (the AI response area). ### Changed

- **initColumnLayout defensive tests** (`test/unit/popup.spec.js`): 7 tests verifying null-guard behavior for all missing DOM scenarios.

- **Popup Logic** (`popup/popup.js`): Removed legacy `renderPresetList()`, `bindDragEvents()`, all HTML5 drag-and-drop event handlers (`dragstart`, `dragover`, `drop`, `dragend`). Wired the `customSelect` API for preset rendering, selection, reorder, and inline actions. Added startup stale-binding cleanup for robustness.
- **Manifest** (`manifest.json`): Version bumped to 1.9.0.
- **Integration Tests**: Updated all Playwright selectors to target the new combobox DOM structure. 71 tests passing across all feature specs.

### Fixed
- **Content Script Overlay Initialisation** (`content/content-script.js`): Fixed `PresetOverlay.start()` execution order to ensure `updateActiveId()` is called only after the overlay `<select>` DOM element is present in the document.
- **Cross-tab overlay state contamination**: The `chrome.storage.onChanged` listener in the content script now derives the overlay's selected ID from the tab's own isolated state (`currentChatUuid`, `chatPresetMap`, and `pendingPresetId`) instead of blindly applying the global `activePresetId` from storage.
- **Popup startup stale-binding cleanup**: `popup.js` now cleans up `chatPresetMap` entries whose referenced preset no longer exists on startup, preventing orphaned bindings from persisting across reloads.
- **StorageManager sync quota fallback** (`utils/storage-manager.js`): `savePromptPresets()` now uses `_set()` for the PRESET_INDEX write, ensuring graceful fallback to local storage when `chrome.storage.sync` exceeds `MAX_WRITE_OPERATIONS_PER_MINUTE`. Added null-safe error message extraction (`syncError?.message ?? 'Unknown error'`).
- **Modal button duplication** (`popup/popup.js`): Added `this.actionsEl.innerHTML = ''` in both `Modal.prompt()` and `Modal.confirm()` before building buttons, preventing stale button accumulation on re-entrant calls. Added `Modal.dismissActive()` to programmatically dismiss modals when a preset is selected from the dropdown.
- **Unhandled promise rejections in input handlers** (`popup/popup.js`): Wrapped async `input` event listeners for `promptInput` and `globalDefaultPromptInput` in try/catch blocks; storage errors now log a warning instead of becoming uncaught rejections.
- **initColumnLayout null safety** (`popup/popup.js`): Added 5 defensive guard clauses (`.container`, `.column`, `.card--header`, `.card--flex`, textarea elements) with early return to prevent TypeError crashes when DOM elements are missing.

## [1.8.0] - 2026-05-14

### Added
- **Inline Preset Selector Overlay**: A floating `<select>` dropdown now appears centered on the chat title bar (`div._2be88ba`) on the DeepSeek page, allowing users to switch prompt presets without opening the popup. The overlay is positioned absolutely within the title bar with a dark semi-transparent style.
- **Bidirectional Sync with Popup**: Changes made via the overlay immediately propagate to the popup (and vice versa). The overlay listens to `chrome.storage.onChanged` for preset list changes and `ACTIVE_PRESET_ID` updates, and the popup's `ACTIVE_PRESET_CHANGED` message directly updates the overlay's selected value.

## [1.7.2] - 2026-05-14

### Fixed
- **Conversation switching breaks UUID-to-preset binding**: Fixed a regression where switching between conversations would always lose the preset binding, falling back to the empty state. The `handleChatChange()` function in the content script was reading the retired `promptPresets` storage key to verify binding validity — this key was removed during the v1.7.0 schema migration, causing every binding to be incorrectly identified as stale and deleted. Now uses `StorageManager.getSettings()` which correctly resolves presets from the new per-key schema (`dsPresetIndex` + `dsPreset_<id>`).

## [1.7.1] - 2026-05-14

### Fixed
- **Stale prompt prefix after editing preset content**: Fixed a regression where editing a prompt preset's content in the popup and then sending a message directly would inject the old content. The `chrome.storage.onChanged` listener in the content script now correctly detects changes to the new per-preset storage keys (`dsPresetIndex` and `dsPreset_<id>`), triggering `updatePromptPrefixFromBinding()` immediately instead of only reacting to the retired `promptPresets` key.
- **Sidebar auto-hide not collapsing after dropdown menu interaction**: Fixed a regression where moving the mouse out of a floating/dropdown menu (`ds-dropdown-menu ds-elevated`) without re-entering the sidebar would not trigger sidebar collapse. The bug was caused by an inverted `!isCollapsed()` guard in the `onLeave` handler that prevented collapse when the sidebar was already expanded.

## [1.7.0] - 2026-05-13

### Added
- **Per-Preset Individual Storage Keys**: Refactored `StorageManager` to store each prompt preset under its own unique key (`dsPreset_<id>`) instead of a single giant array. This bypasses the 8KB per-item limit of Chrome Sync storage, allowing for significantly larger and more numerous prompt presets.
- **Plan A: Local Authoritative Tracking**: Implemented a "Local Authoritative" mechanism to fix a critical sync-over-local bug. When sync fails (e.g., due to quota limits), the affected keys are marked as local-authoritative. During subsequent reads, the extension now correctly prioritizes the newer local data over stale cloud data for these specific keys, ensuring no data loss when exceeding sync quotas.

### Changed
- **Storage Schema Evolution**: Introduced `dsPresetIndex` to maintain the ordered list of preset IDs and `dsLocalAuth` for tracking local-authoritative keys. The old `promptPresets` key is automatically migrated to the new format upon first load.

## [1.6.6] - 2026-05-12

### Fixed
- **Cross-conversation prompt inheritance on new tab**: Fixed a bug where opening a new conversation page (`https://chat.deepseek.com/`) in a new tab would inherit `promptPrefix` from the global `activePresetId` of the previous tab. The content script no longer preloads `promptPrefix` via `getActivePromptContent()` in `initSettings()`; `promptPrefix` is now fully derived from the current chat's UUID binding through `handleChatChange()`. The no-UUID branch of `handleChatChange()` now unconditionally clears `promptPrefix`, fixing the case where the initial `currentChatUuid` was already `null` and the previous guard was skipped.
- **Unintended auto-bind on manual conversation switch**: Fixed a bug where selecting a preset on a new conversation page and then manually switching to another (previously unbound) conversation would incorrectly auto-bind that existing conversation to the pending preset. Auto-bind now requires an explicit `awaitingNewChatUuid` flag that is only set when the user actually triggers a message send (via Enter key or send button click) on a no-UUID page. The flag is consumed by `handleChatChange()` and self-clears after 5 seconds to avoid stale-flag pollution after failed sends.

### Changed
- **Auto-bind trigger condition tightened**: The auto-bind branch in `handleChatChange()` (previously triggered on any `no-UUID → UUID` transition) now requires `awaitingNewChatUuid === true`, ensuring every conversation maintains an independent binding relationship as mandated by `TODO.md`.

## [1.5.1] - 2026-05-07

### Fixed
- **Popup UI padding consistency**: Removed redundant `margin-bottom` from `header` and `.checkbox-group` elements. All vertical spacing between rows is now uniformly controlled by the card's `gap: 12px`, eliminating double-spacing inconsistencies.
- **New chat input box not affected by input width setting**: Added `.aaff8b8f` as a standalone CSS selector (without the `._871cbca` parent requirement) in the `InputWidth` module. New conversation pages use a different DOM structure where the input area class is not nested under `._871cbca`, causing the previous selector `._871cbca .aaff8b8f` to not apply.
- **Input width not clamped by chat width**: Fixed two call sites in `InputWidth` that used `this.percent` directly instead of `this.getEffectivePercent()`: the `enable()` method and the `MutationObserver` re-application callback. The effective percent correctly caps the input width to the chat width when chat width adjustment is enabled and smaller.
- **Cross-tab prompt group pollution**: Implemented per-tab preset isolation. The content script no longer blindly updates `promptPrefix` from the global `ACTIVE_PRESET_ID` storage key. Instead, the popup sends `ACTIVE_PRESET_CHANGED` messages directly to the active tab's content script. Each tab independently tracks its `pendingPresetId` and derives `promptPrefix` from its own UUID binding via `updatePromptPrefixFromBinding()`. Auto-bind on first message also uses per-tab state rather than global storage.
- **Sidebar collapse on dropdown menu hover**: When the mouse moves from the sidebar into a floating/dropdown element (e.g., context menu rendered via React portal), the sidebar now stays expanded until the user dismisses the dropdown. Previously, leaving the sidebar's bounding box to interact with an overflow dropdown would trigger the collapse timer, making dropdown items inaccessible. The fix uses a capture-phase `mouseover` listener on `document` that detects floating elements by class patterns (`ds-floating*`, `ds-dropdown*`, `ds-elevated`), cancels the collapse timer, and monitors the dropdown's `mouseleave` for cleanup.

### Changed
- **Storage listener in content script**: Replaced reactive `ACTIVE_PRESET_ID` listener with message-based per-tab communication. Only `PROMPT_PRESETS` and `CHAT_PRESET_MAP` changes are monitored from storage; the content script recalculates `promptPrefix` based on the current chat's UUID binding.
